# 📊 서울시 상권 빅데이터 분석 및 창업 추천 파이프라인

## 1. 문제 정의 및 프로젝트 배경

### 자영업 시장의 포화와 생존 위기
현재 대한민국 자영업 시장은 구조적인 포화 상태와 심각한 생존 위기에 직면해 있다. 최근 통계에 따르면 전체 취업자 중 자영업자 비율은 2023년 기준 **23.2%**로, OECD 평균인 15.6%와 비교하면 크게 높은 수준이다. 특히 과도한 경쟁과 경기 침체가 맞물리면서 폐업률은 심각한 수준에 이르렀다. 국세청에 따르면 2024년 폐업 신고 사업자는 **100만 명을 돌파**했으며, 이는 전체 사업자 10곳 중 1곳이 문을 닫고 있다는 것을 의미한다.

### 폐업 위험을 높이는 정보의 부재
이러한 폐업 100만 명 시대에 가장 큰 문제는 창업자들이 객관적인 상권 지표 없이 감이나 단편적인 정보에 의존하여 창업 입지와 업종을 결정한다는 점이다. 특정 지역의 유동인구만 믿고 창업하거나, 동종 업계의 포화도 및 실제 고객의 소비 패턴을 파악하지 못한 채 진입하는 것은 폐업 위험을 높이는 주된 원인이 된다.

### 프로젝트 목표
본 프로젝트는 빅데이터 파이프라인을 구축하여 이러한 창업 의사결정 문제를 해결하고자 한다. 이를 위해 서울 열린데이터광장에서 제공하는 공공데이터를 기반으로 유동인구 대비 실제 매출액이 높고 폐업률이 낮은 최적의 창업 지역과 업종 추천 지표를 도출하는 것이 최종 목표이다. 이를 통해 예비 창업자들이 직감이 아닌 실제 데이터에 근거한 합리적인 선택을 할 수 있도록 지원하고자 한다.

### 핵심 분석 질문 (임시)
1. 특정 업종의 매출이 높은 상권들의 주요 소비 인구 패턴은 어떠한가?
2. 최근 3년간 폐업률이 낮으면서 꾸준한 매출을 유지하는 상권과 업종의 조합은 무엇인가?
3. 매출, 유동인구 등을 결합하여 창업 성공 가능성이 높은 최적의 지역 및 업종은 무엇인가?

## 2. 사용 데이터셋
서울 열린데이터광장의 상권분석서비스 핵심 데이터 활용

1. **추정매출-상권**
   - **핵심 데이터:** 당월 매출 금액, 당월 매출 건수, 연령대별/성별/시간대별 매출 금액
   - **설명:** 상권 및 업종별로 발생한 총매출 규모와 결제 건수를 집계한 데이터. 상권 내 주 소비층의 구매 패턴을 분석하는 핵심 지표로 사용.

2. **점포-상권**
   - **핵심 데이터:** 점포 수, 프랜차이즈 점포 수, 개업 점포 수, 폐업 점포 수
   - **설명:** 특정 상권 내 업종별 점포 수와 프랜차이즈 비율을 보여주는 데이터. 개폐업 점포 수 추이를 통해 업종별 경쟁 강도와 생존 난이도를 평가하는 데 활용.

3. **길단위인구-상권**
   - **핵심 데이터:** 총 유동인구 수, 연령대별/성별/시간대별 유동인구 수
   - **설명:** 특정 상권 영역의 유동인구 수를 측정한 데이터. 매출 데이터와 결합하여 특정 상권의 객관적인 수요 규모를 파악하는 데 활용.

4. **영역-상권**
   - **핵심 데이터:** 상권 코드, 상권 코드 명, 상권 구분 코드 명
   - **설명:** 상권 코드를 실제 상권 이름으로 매핑해 주는 데이터. 병합 후 가독성을 높이는 데 활용.

## 3. 기술 스택

### Language
- **Python:** 데이터 수집 스크립트 작성 및 파이프라인 전체 흐름 제어 

### Data Storage
- **HDFS:** 수집된 대용량 CSV 데이터를 파티션 구조로 분산 저장

### Data Processing & Analysis
- **Spark (Spark SQL):** 불필요한 열 제거 및 다중 데이터셋 병합(Join) 수행
- **Hive (HiveQL):** HiveQL 쿼리를 실행하여 다차원 통계치 산출

### Data Visualization
- **Pandas & Matplotlib / Seaborn:** 최종 통계 데이터를 정형화하고, 직관적인 차트로 시각화

## 4. 구현 계획

### 1단계: Data Collection 
- Python 스크립트를 활용하여 서울시 상권 데이터 자동 호출
- 스트리밍 형태의 수집 파이프라인 구현

### 2단계: Storage
- 수집 단계에서 분할된 데이터를 HDFS에 순차적 적재

### 3단계: Processing & Analysis 
- Spark를 이용해 결측치 및 불필요한 열을 필터링하고, 하나의 테이블로 병합
- Hive를 이용해 여러 지표를 결합한 최종 통계(다차원 상권 추천 지수) 도출

### 4단계: Result Presentation 
- 최종 통계 데이터(ex. 최종 상권 추천 순위)를 Pandas로 정형화
- Matplotlib, Seaborn을 활용하여 시각화 차트 제공
