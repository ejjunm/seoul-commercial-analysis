# 📊 서울시 상권 빅데이터 분석 및 2030 청년 창업 추천 파이프라인

서울 열린데이터광장 상권 공공데이터를 HDFS·Spark·Hive로 처리하고, 거시적 상권 분석 및 XGBoost로 2030 청년에게 안착 가능성이 높은 상권×업종을 추천하는 빅데이터 파이프라인.

Repo: https://github.com/ejjunm/seoul-commercial-analysis

---

## 1. 문제 정의 및 프로젝트 배경

### 자영업 시장의 포화와 생존 위기
현재 대한민국 자영업 시장은 구조적인 포화 상태와 심각한 생존 위기에 직면해 있다. 최근 통계에 따르면 전체 취업자 중 자영업자 비율은 2023년 기준 23.2%로, OECD 평균인 15.6%와 비교하면 크게 높은 수준이다. 특히 과도한 경쟁과 경기 침체가 맞물리면서 폐업률은 심각한 수준에 이르렀다. 국세청에 따르면 2024년 폐업 신고 사업자는 **100만 명을 돌파**했으며, 이는 전체 사업자 10곳 중 1곳이 문을 닫고 있다는 것을 의미한다.

### 폐업 위험을 높이는 정보의 부재
이러한 폐업 100만 명 시대에 가장 큰 문제는 창업자들이 객관적인 상권 지표 없이 감이나 단편적인 정보에 의존하여 창업 입지와 업종을 결정한다는 점이다. 특정 지역의 유동인구만 믿고 창업하거나, 동종 업계의 포화도 및 실제 고객의 소비 패턴을 파악하지 못한 채 진입하는 것은 폐업 위험을 높이는 주된 원인이 된다.

### 프로젝트 목표
본 프로젝트는 빅데이터 파이프라인을 구축하여 이러한 창업 의사결정 문제를 해결하고자 한다. 이를 위해 서울 열린데이터광장에서 제공하는 공공데이터를 기반으로 유동인구 대비 실제 매출액이 높고 폐업률이 낮은 최적의 창업 지역과 업종 추천 지표를 도출하는 것이 최종 목표이다. 이를 통해 예비 창업자들이 직감이 아닌 실제 데이터에 근거한 합리적인 선택을 할 수 있도록 지원하고자 한다.

### 핵심 분석 질문
| | 질문 | 관점 |
|---|---|---|
| **Q1** | 서울 자치구별 시장의 규모·실속·특화 순위는 어떻게 다른가 | 거시 |
| **Q2** | 8분기 추세상 검증된 메가 상권과 신흥 성장 중인 라이징 상권은 어디인가 | 성장 |
| **Q3** | 2030 청년이 진입하면서도 안착할 수 있는 상권×업종 조합은 무엇인가 | 추천 |

## 2. 사용 데이터셋
- **출처:** 서울 열린데이터광장 (상권분석서비스)
- **기간:** 2024 Q1 ~ 2025 Q4 (총 8개 분기, 약 145.3MB)
- **대상:** 생활밀착 외식·소매 10개 업종 (한식음식점, 중식음식점, 일식음식점, 양식음식점, 제과점, 패스트푸드점, 치킨전문점, 분식전문점, 호프-간이주점, 커피-음료)


1. **추정매출-상권**
   - **핵심 데이터:** 당월 매출 금액, 당월 매출 건수, 연령대별/성별/시간대별 매출 금액
   - **설명:** 상권 및 업종별로 발생한 총매출 규모와 결제 건수를 집계한 데이터. 상권 내 주 소비층의 구매 패턴을 분석하는 핵심 지표로 사용.

2. **점포-상권**
   - **핵심 데이터:** 점포 수, 개업 점포 수, 폐업 점포 수
   - **설명:** 특정 상권 내 업종별 점포 수와 프랜차이즈 비율을 보여주는 데이터. 개폐업 점포 수 추이를 통해 업종별 경쟁 강도와 생존 난이도를 평가하는 데 활용.

3. **길단위인구-상권**
   - **핵심 데이터:** 총 유동인구 수, 연령대별/성별/시간대별 유동인구 수
   - **설명:** 특정 상권 영역의 유동인구 수를 측정한 데이터. 매출 데이터와 결합하여 특정 상권의 객관적인 수요 규모를 파악하는 데 활용.

4. **영역-상권**
   - **핵심 데이터:** 상권 코드, 상권 코드 명, 상권 구분 코드 명, 엑스/와이 좌표
   - **설명:** 상권 코드를 실제 상권 이름으로 매핑해 주는 데이터. 병합 후 가독성과 지도 시각화에 활용.

## 3. 기술 스택

| 기술 | 역할 | 적용 위치 |
|---|---|---|
| **HDFS** | 원본·정제 중간 산출물 저장 | 파이프라인 전 단계 |
| **Apache Spark 2.x** | 원본 데이터 변환·정제·병합 | `preprocess.py`, `run_analysis.py`, `build_dashboard_data.py` |
| **Apache Hive (HQL)** | Master 테이블 SQL 집계 | `analyze_insight.hql` |
| **XGBoost + SHAP** | 청년 안착 적합도 예측 및 변수 해석 | `ml_insight_q3.py` |
| **pandas + pyproj** | 로컬 처리 및 공간 좌표 변환 | `preprocess.py`, `ml_insight_q3.py` |
| **Plotly / Streamlit** | 정적 HTML 및 인터랙티브 대시보드 | `build_dashboard_html.py`, `app.py` |
| **GNU Make** | 6단계 파이프라인 자동화 | `Makefile` |

### 저장 포맷
- **원본** → CSV (`/data/raw`)
- **가공 Master / ML 결과** → Parquet (`/data/processed`)
- **대시보드 요약** → CSV

## 5. 실행 방법

### 5.1 전체 파이프라인 실행

```bash
make all
```

`setup → ingest → preprocess → ml → analyze → dashboard` 순서로 데이터 수집부터 대시보드 요약 데이터 생성까지 자동화. 8분기 데이터를 수집하고 XGBoost 학습까지 거치므로 시간 소요.

### 5.2 대시보드 시각화

대시보드는 정적 HTML과 Streamlit 인터랙티브 두 가지로 제공. 두 가지 모두 `make dashboard` 명령어로 실행된 요약 CSV를 참조하므로, 사전에 `make all` 또는 `make dashboard` 실행 필수

#### (A) 정적 HTML — 고정 IP로 접속

```bash
make dashboard-html
```

`build_dashboard_html.py`가 `dashboard.html`을 생성하고 `8888` 포트로 구동. 브라우저에서 아래 주소 접속 가능.

```text
[http://34.50.49.43:8888/dashboard.html](http://34.50.49.43:8888/dashboard.html)
```
> **Live Preview:** 별도의 환경 세팅 없이, 위 링크를 통해 정적 대시보드 결과물을 즉시 확인 가능.

#### (B) Streamlit 인터랙티브 — 임시 공개 주소로 접속

`app.py`는 WebSocket으로 실시간 통신을 유지. 외부망에서 방화벽 이슈 없이 접속하기 위해 Cloudflare 터널을 활용. (PuTTY 창 2개 필요)

### 5.3 단계별 실행

전체를 다시 돌릴 필요 없이 특정 단계만 점검할 때 사용. 앞 단계 산출물이 HDFS에 있어야 다음 단계가 동작.

```bash
make setup        # Python 3.7 + Spark 환경 및 의존성 설치
make ingest       # 데이터 수집 후 HDFS 적재
make preprocess   # 전처리·병합 → Master(Parquet)
make pipeline     # ingest + preprocess 처리
make ml           # Q3 XGBoost 추천 모델 학습
make analyze      # Q1·Q2 거시 분석 
make dashboard    # 대시보드용 요약 CSV 생성
```

### 5. 4 기타 유틸리티

```bash
make sample       # 데이터 구조 검증용 샘플 생성 (data/sample)
make hdfs-ls      # HDFS raw/processed 적재 현황 확인
make clean        # 임시 파일·샘플 정리
```

## 6. 결과 요약

### Q1. 자치구 거시 분석
규모·실속·특화 세 축에서 1위 자치구가 모두 상이하게 도출. 이는 절대적인 매출 규모가 곧 큰 상권의 효율을 의미하지 않음을 시사

| 기준 | 1위 | 수치 | 메모 |
|---|---|---|---|
| 규모 (총매출) | 강남구 | 69,260억 원 | 2위 중구의 약 1.5배 |
| 실속 (점포당 매출) | 중구 | 1.104억 원 | 규모 1위 강남은 3위로 역전 |
| 특화도 (LQ) | 용산구 양식음식점 | 3.30 | 매출·점포 LQ 모두 3 이상 |

### Q2. 메가 & 라이징 핫플
도심 업무·관광 거점에 몰린 고진입 검증 상권(메가)과 주거·외곽을 중심으로 폭발적으로 팽창 중인 신흥 상권(라이징)을 도출

| 구분 | 상위 상권 (업종) | 지표 |
|---|---|---|
| 메가 | 종각역 (일식) | 상승 총점 97.14 |
| 메가 | 북창동 (한식) | 점포당 14,719만 원, 성장률 39% |
| 라이징 | 은마아파트 (커피-음료) | +35% |
| 라이징 | 성수역 (호프-간이주점) | +30% |


### Q3. 2030 청년 추천
초기 골목상권 3,745건을 8단계로 필터링을 거친 727건의 후보군애서 추천 순위를 도출

| 순위 | 상권 (자치구) | 업종 | 추천점수 |
|---|---|---|---|
| 1 | 방배초등학교 (서초) | 양식음식점 | 0.744 |
| 2 | 신월2동주민센터 (양천) | 커피-음료 | 0.737 |
| 3 | 성서초등학교 (마포) | 양식음식점 | 0.682 |
| 4 | 천호역 1번 (강동) | 커피-음료 | 0.653 |
| 5 | 연희초등학교 (서대문) | 분식전문점 | 0.643 |

거시 분석(Q1, Q2)이 강남·종로 등 거대 자본 중심의 도심에 쏠렸던 것과 달리, 추천 상위는 서초·양천·마포·강동 등 주거·생활밀착 상권으로 고르게 도출. 이는 모델이 임대료와 권리금이 폭등한 핵심 상권을 걸러내고, 실질적인 생존 확률이 높은 진입 가능 상권을 성공적으로 타겟팅

```
추천점수    = 안착 적합도 × 진입 가능성
진입 가능성 = (1 − 경쟁밀도) × 0.7 + 유동인구 적정도 × 0.3
```

XGBoost 모델 성능은 5-Fold CV AUC 0.548을 기록하였으며, 미래를 단정하는 예측기가 아니라 예측 후보를 줄이는 도구로 사용.

## 7. 한계 및 개선 방향
- **공공데이터의 한계**: 임대료·권리금·창업자 역량·마케팅 등 창업의 핵심 변수의 데이터 부재 
- **고정 임계값 설정의 한계**: MZ 35%·유동인구 65% 최적점 등 임계값이 하드코딩돼 있어, 추후 사용자 맞춤 조정이 가능하도록 개선 예정
- **외부 도메인 융합**: 상권별 임대료, SNS 해시태그와 같은 외부 API를 연동해, 실시간 트렌드와 임대료가 반영된 추천 지수 개발
